属性 作用
activeClassName 当活动时给出元素的类。默认给定类是活动的。这将与className支持相结合。

Route


仅当位置匹配时才呈现的React组件
这允许方便的在线呈现和包装，而不需要上述的不期望的重新安装。您可以使用组件支持为您创建一个新的React元素，而不必在位置匹配时传入要调用的函数。
**
有时您需要渲染路径是否匹配该位置

属性 作用
path path-to-regexp理解的任何有效的URL路径
exact 当为true时，仅当路径与location.pathname完全匹配时才匹配。
strict 当为true时，具有尾部斜杠的路径将仅与具有尾部斜杠的location.pathname匹配
location 元素尝试将其路径与当前历史记录位置（通常是当前浏览器URL）进行匹配

StaticRouter

一个从不改变位置的。

当用户实际上没有点击时，这在服务器端渲染场景中很有用，因此该位置从未实际改变。因此，名称：static。当您只需插入一个位置并在渲染输出上作出断言时，它也可用于简单的测试

属性 作用
basename 所有位置的基本URL。正确格式化的基础名称应该有一个主要的斜杠，但没有尾部斜线。
location(string,object) 服务器接收到的URL，可能是节点服务器上的req.url。
context 一个简单的JavaScript对象。在渲染过程中，组件可以向对象添加属性以存储有关渲染的信息
